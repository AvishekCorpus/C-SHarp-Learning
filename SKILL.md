# Code Explanation Skill

## Description
This skill provides detailed explanations of code snippets in the ShipmentLearning C# project, focusing on syntax, semantics, purpose, and project context. It's workspace-scoped for this project and follows a full multi-step workflow to ensure thorough understanding.

## Steps
1. **Identify the Snippet**: Select the code lines or block to explain (e.g., from Program.cs).
2. **Break Down Syntax**: Explain keywords, types, operators, and structure using C# language rules.
3. **Explain Semantics**: Describe what the code does at runtime, including data flow and behavior.
4. **Provide Purpose**: Clarify why the code exists, its role in the program, and any fixes or improvements (e.g., from GAP comments).
5. **Add Context**: Reference related parts of the project (e.g., Parcel class, Main method) and how it integrates.

## Decision Points
- **Complexity Check**: If the snippet involves advanced features (e.g., LINQ, delegates), include simplified examples or analogies.
- **Project Relevance**: Always tie back to the ShipmentLearning app's goals (parcel management, input validation).
- **User Level**: Assume basic C# knowledge but explain jargon; if unclear, suggest reading related docs.

## Quality Criteria
- **Accurate**: Explanations must align with C# specifications and the code's actual behavior.
- **Concise**: Keep details focused; avoid rambling.
- **Complete**: Cover all aspects without omitting key points.
- **Actionable**: Help the user understand how to modify or debug the code if needed.
